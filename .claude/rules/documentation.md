# Documentation Rules

> Canonical agent-neutral text: `core/standards/documentation.md`. Keep both
> in sync.

Documentation should answer:

- What exists now?
- Why does it exist?
- How is it used?
- What are the constraints and risks?
- How is it verified?
- How can it be rolled back or operated?

When code changes alter behavior, architecture, environments, CI/CD, deployment, secrets, rollback, observability, or operational flow, update the relevant docs.

Prefer concise documents with clear sections over long generic explanations.
