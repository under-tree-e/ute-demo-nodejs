# UTE Frontend Rules

- Follow the project’s existing framework and style conventions.
- Do not migrate framework/style architecture as a side effect of a feature.
- For UI changes, handle loading, error, empty, and disabled states when relevant.
- Keep accessibility in mind: labels, keyboard navigation, semantic elements, contrast.
- Comment only non-obvious business logic or tricky UI state, not ordinary code.
